# Contributing

First of all, thank you for taking the time to contribute!  
Contributions are what make the open-source community such a great place to learn, inspire, and build together.  
Every contribution, no matter how small, is truly appreciated.

## Bug reports

When reporting a bug, please make sure your report is:

- **Reproducible** – include clear steps to reproduce the issue
- **Specific** – provide as much detail as possible (version, environment, logs, etc.)
- **Unique** – avoid duplicating existing open issues
- **Focused** – report one bug per issue

Please also make sure to follow this project’s **Code of Conduct**.

## Style guidelines

You can use `markdownlint-cli` to check for common Markdown style and formatting issues before submitting documentation changes.
